Release Notes 1.3.2 - Feb 9th 2019¶

In this release, we made the following improvements

  • Overall infrastructure
    • added support for instrument simulation without the pyre infrastructure. Some examples of using this new capability can be found in the training notebooks
    • made the histogram-merging step (after simulation) faster
    • added support for gcc 7 compiler
    • added support for boost 1.69
    • added a tool to convert mcstas instrument to non-pyre python script. $ mcvine mcstas convertinstrument
  • Sample assembly
    • kernel orientation. Example
    • new shapes: pyramid and cone. See shapes
    • shape validation: the command $ mcvine sampleassembly check {filename} now checks whether shapes in the sample assembly overlap
    • warn about max_multiplescattering_loops_among_scatterers parameter if not set conservatively during sampleassembly check
    • xml: support union and intersection for more than two shapes
    • xml: support spec of rotation as axis and angle. Example
    • added support for cif files for specification of material structure. Example
    • added a function to create diffraction peaks python file from material structure. Example
    • support of shape specification using yaml. Example
    • added a S(Q) kernel. Example
    • (developer) introduced boundingbox. BoundingBoxMaker
  • Other
    • added a command to print mcvine version: $ mcvine version
    • fixed various bugs

Please refer to related tickets for more details.
